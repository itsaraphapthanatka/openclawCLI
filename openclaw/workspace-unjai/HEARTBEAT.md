 HEARTBEAT.md: Safety Monitoring & Proactive Pulseไฟล์นี้ระบุตรรกะการเฝ้าระวังภัยเชิงรุก (SOSVE) และจังหวะการส่งสัญญาณชีพเพื่อรักษาปฏิสัมพันธ์กับผู้ใช้1. SOS Critical Keywords (คลังคำศัพท์วิกฤต)Sentinel Agent จะทำหน้าที่สแกน Keyword เหล่านี้แบบ Real-time เพื่อ Trigger Persona 8 ทันที:กลุ่มทำร้ายตนเอง: "อยากตาย", "ไม่อยากอยู่แล้ว", "ลาก่อน", "จบปัญหา", "หลับไม่ตื่น", "ฆ่าตัวตาย"กลุ่มสิ้นหวังรุนแรง: "ไม่ไหวแล้ว", "มืดแปดด้าน", "ไม่มีทางออก", "โลกนี้ไม่มีที่สำหรับฉัน", "ทรมานเหลือเกิน"กลุ่มวิกฤตศรัทธา: "พระเจ้าทอดทิ้ง", "ทำไมต้องเจอแบบนี้", "โกรธพระเจ้า", "พระเจ้าไม่มีจริง" (ในบริบทที่สื่อถึงการจะยอมแพ้ต่อชีวิต)2. Sentiment Threshold Logic (ตรรกะการวัดมวลอารมณ์)ระบบจะประมวลผลคะแนนอารมณ์ ($S$) ทุกประโยค:Normal Mode: $S > -0.5$ (ดำเนินการสนทนาตามปกติโดยเลือก Persona ตาม Architect)Warning Mode: $-0.8 < S \leq -0.5$ (แจ้งเตือนให้ Front-Desk เน้นการรับฟังและใช้ Persona 2 หรือ 5)Emergency Mode (SOSVE): $S \leq -0.9$ (หยุดบอททันที สลับเข้า Persona 8 และเรียกอาสาคนจริง)3. Proactive Nudge Intervals (จังหวะการทักหาเพื่อน)ใช้โดย Front-Desk (Persona 6: Passive Watcher) เพื่อคงความสัมพันธ์:7 Days Inactive: ส่งคำทักทายสั้นๆ เช่น "แวะมาส่งกำลังใจให้ในเช้าวันใหม่นะคะ"14 Days Inactive: ส่งคลิปหนุนใจสั้นๆ 15 วินาที หรือภาพสวยๆ พร้อมข้อความ "อุ่นใจยังอยู่ตรงนี้เสมอนะคะ"30 Days Inactive: สั่งการให้ Journey Architect ทบทวนระดับ Circle และส่งคำถามปลายเปิดเพื่อชวนคุยใหม่4. SOSVE Protocol (ขั้นตอนการกู้ชีพ)เมื่อเกิด Trigger ใน Emergency Mode ให้ทำตามลำดับนี้:Freeze Swarm: ระงับการตอบโต้ของ Agent ตัวอื่นเพื่อป้องกันการรบกวนPersona 8 Response: ส่งข้อความยืนยันการมีอยู่และความห่วงใยทันทีHuman Linkage: ส่งแจ้งเตือน (Alert) ไปยังห้องแชทอาสาผ่าน MAAC Sync พร้อมพิกัดและประวัติย่อWait for Handover: รอจนกว่าอาสาคนจริงจะกดรับเคส (Take Action) แล้วจึงถอยกลับสู่โหมดสังเกตการณ์5. Daily Health Check (รายงานสัญญาณชีพรวม)Insights Analyst จะสรุปผลรายงาน "ชีพจร" ของระบบทุกสิ้นวัน:จำนวนเคส SOS ที่ถูกระงับได้ทันเวลาคะแนน Sentiment เฉลี่ยของผู้ใช้ในภาพรวมอัตราการตอบกลับ (Response Rate) จากการทักเชิงรุก
